## 1. 프로젝트 개요

### 아이디어

투자에서 수익을 내기 어려운 가장 큰 이유 두 가지는 **방향 예측의 불확실성**과 **이벤트 발생 타이밍의 사후성**이다.  
이 프로젝트는 이 두 한계를 정면으로 인식하고, 다음 질문에서 출발한다:

> "방향을 맞추지 않고도, 이벤트가 발생했을 때 확률적으로 수익이 나는 구조를 만들 수 있는가?"

이를 위해 두 개의 독립 시스템을 구축했다:

| 시스템 | 경로 | 목적 |
|---|---|---|

| **macro-portal** | `macro-portal/` | 거시 이벤트 감지 + 위기 시그널 시각화 |

## 2. 핵심 문제의식

### 2-1. 방향 예측의 한계

ETF 스트래들 전략 (`research_straddle.py`) 연구에서 확인:
- 특정 이벤트(호르무즈 봉쇄, 반도체 수출 규제 등)가 발생하면 해당 섹터 ETF는 단기에 큰 변동
- 문제는 **이벤트가 발생한 후** 방향(상승/하락)을 알기 때문에 사후 진입은 의미 없음
- 따라서 방향에 베팅하는 전통적 접근은 이 구조에서 작동하지 않음

### 2-2. 스트래들 구조의 아이디어

- **롱 + 숏 동시 진입** (각각 +10% 목표로 청산)
- 어느 방향으로 크게 움직여도 한쪽이 수익
- 수익 구조: `combined_pnl = (long_pnl + short_pnl) / 2`
- 이론적으로 변동성이 충분히 크면 **드리프트와 무관하게 E[PnL] > 0**

### 2-3. 선제 감지의 필요성

스트래들이 수익을 내려면 **이벤트 발생 전에 포지션을 잡아야 한다**.  
여기서 핵심 질문이 생긴다: "변동성이 폭발하기 전에 미리 감지할 수 있는가?"

→ 이것이 **Vol Anomaly Monitor** + **5-Layer Ontology 분석 프레임워크** 개발의 동기.

### 4-2. Layer 1 — GARCH(1,1)

**파일**: `macro_research/garch_estimator.py`

**목적**: 옵션 체인 없이 내재변동성(IV)의 대리변수를 추정한다.

#### 수식

GARCH(1,1) 모델:

```
σ²_t = ω + α·ε²_(t-1) + β·σ²_(t-1)
```

- `ω` : 장기 평균 분산 (intercept)
- `α` : 최근 충격(ε²)이 분산에 미치는 즉각 반응 계수
- `β` : 전일 조건부 분산의 지속성 계수
- `α + β < 1` : 분산이 장기 평균으로 회귀하는 조건 (mean-reversion)

연율화:

```
σ_GARCH_annual = σ_GARCH_daily × √252
```

#### VRP (Volatility Risk Premium) — 두 가지 정의

**진짜 VRP (2026-06-08~, `iv_extractor.py`)**:
```
VRP_true = ATM_IV  −  RV_20d   (Q vs P 갭)
```
- `ATM_IV` : Alpaca options snapshot 에서 ATM 콜+풋 implied volatility 평균 (연환산)
- `RV_20d` : 과거 20일 일별 수익률 표준편차 × √252
- 이게 *옵션 시장이 받아내는 값 vs 실제 실현될 값* 의 갭 = 정의에 충실한 VRP

**Legacy VRP (σ_GARCH 기반, 분석 보존용)**:
```
VRP_legacy = σ_GARCH  −  RV_20d
```
- σ_GARCH 는 *물리측도(P)* 모델 예측 — IV(Q) 가 아님
- "옵션이 비싸다" 측정에 부적합. 변동성 모멘텀 신호일 뿐
- `rule_vol_overpriced` 는 더 이상 이 값을 보지 않음 (2026-06-08 수정)

| VRP_true 부호 | 의미 | 전략 시사 |
|---|---|---|
| > +2% | 옵션 IV > 실현 RV → 옵션 비쌈 | 숏 볼 후보 (현재는 STRADDLE ×0.5 페널티만, 트랙 미구현) |
| ≈ 0 | 균형 | 중립 |
| < -5% | IV < RV → 옵션 과소평가 | 꼬리 리스크 경고 |

#### Vol Monitor에서의 CAPM IV

Vol Monitor에서는 GARCH 대신 CAPM 기반 IV 추정을 사용한다 (실시간 yfinance 데이터만으로 계산 가능):

```
CAPM_IV = sqrt( (β × VIX/100)² + σ_idio² )
```

- `β` : 63일 롤링 공분산 / SPY 분산
- `σ_idio` : CAPM 잔차의 32일 EWMA 표준편차 (설명 안 되는 고유 변동성)
- `VIX/100` : 시장 내재변동성 (소수점 단위)

`CAPM_IV`가 높다는 것은 해당 섹터의 **베타 노출** 또는 **고유 불확실성** 중 하나 이상이 크다는 의미다.

### 4-3. Layer 2 — EVT / GPD (Extreme Value Theory)

**파일**: `macro_research/tail_analysis.py`

**목적**: 정규분포가 과소평가하는 극단적 손실(fat tail)을 정량화한다.

#### 왜 정규분포가 부족한가

금융 수익률은 정규분포보다 꼬리가 두껍다(leptokurtosis). 표준 VaR(99%)는 이 꼬리 질량을 과소평가한다.

EVT는 꼬리 부분만 별도로 모델링한다.

#### GPD 적합 방법

1. 손실 분포(= -수익률)의 상위 **10% 초과분**을 추출
2. 초과분에 GPD(Generalized Pareto Distribution) 적합:

```
F(y) = 1 - (1 + ξ·y/σ)^(-1/ξ)    (ξ ≠ 0)
```

- `u` : 임계값 (10% 분위수)
- `ξ` (xi, shape) : 꼬리 지수
- `σ` (sigma, scale) : 스케일 파라미터

#### ξ 해석

| ξ 값 | 꼬리 형태 | 의미 |
|---|---|---|
| ξ < 0 | 얇은 꼬리 | 손실에 상한 존재 (드문 경우) |
| ξ = 0 | 지수 꼬리 | 정규·대수정규에 가까움 |
| ξ > 0 | **두꺼운 꼬리 (fat tail)** | 극단 손실 확률이 높음 |
| ξ > 0.5 | **분산이 무한대** | 표준 VaR 자체가 무의미 |
| ξ > 1.0 | **평균조차 무한대** | 모든 모멘트 계산 불가 |

#### EVT 기반 VaR(99%), ES(99%) 수식

```
VaR(99%) = u + (σ/ξ) × ((ζ_u / 0.01)^ξ − 1)

ES(99%)  = (VaR(99%) + σ − ξ·u) / (1 − ξ)
```

여기서 `ζ_u = n_exceed / n_total` (임계값 초과 비율).

#### Cornish-Fisher 수정 VaR

GARCH나 GPD 없이도 왜도·첨도로 VaR를 보정:

```
z_cf = z + (z²-1)·S/6 + (z³-3z)·K/24 − (2z³-5z)·S²/36

VaR_CF = −(μ + σ × z_cf)
```

- `z` : 정규분포 1% 분위수 (= -2.326)
- `S` : 왜도 (skewness)
- `K` : 초과 첨도 (excess kurtosis)

**CF VaR / Normal VaR 비율**이 2.5배를 초과하면 → 정규 VaR 모델이 절반 이하의 자본을 요구한다는 경고.  
온톨로지 규칙 `normal_var_inadequate`가 이 조건에서 MONITOR 시그널을 발생시킨다.

#### 분위수 회귀 (Quantile Regression)

꼬리 비대칭성을 정량화:

```
R_τ = α + β(τ) × dX   (τ = 0.01, 0.05, ..., 0.99)
```

**꼬리 비대칭도** = `β(τ=0.05) − β(τ=0.95)`

- 음수 → 하락장에서 매크로 충격이 더 크게 전달됨 (left tail이 steeper)
- 온톨로지 규칙 `crash_vulnerable`의 핵심 입력

### 4-4. Layer 3 — Copula / 꼬리 의존성

**파일**: `macro_research/tail_analysis.py`

**목적**: 정상 상관관계는 위기 시 상승하므로 믿을 수 없다. 꼬리 구간에서만의 동시 이동 확률을 계산한다.

#### 개념: 왜 코퓰라인가

피어슨 상관계수는 선형·정규 가정 하의 **평균적 의존성**을 측정한다.  
위기(하락 5% 이하)에서는 평소 무관했던 섹터들이 함께 폭락한다 — 이것이 코퓰라가 포착하는 **꼬리 의존성**이다.

이 프로젝트에서는 매개변수적 코퓰라 함수를 가정하지 않고, **경험적 꼬리 의존성**을 직접 계산한다.

#### 수식

```
λ_L(u) = P(X < Q_X(u)  |  Y < Q_Y(u))   ← 하방 꼬리 의존성 (동시 폭락)
λ_U(u) = P(X > Q_X(1-u) | Y > Q_Y(1-u)) ← 상방 꼬리 의존성 (동시 급등)
```

구현 (u = 0.05, 하위 5%):

```python
lambda_L = mean( (X < Q_X(0.05)) & (Y < Q_Y(0.05)) ) / 0.05
lambda_U = mean( (X > Q_X(0.95)) & (Y > Q_Y(0.95)) ) / 0.05
```

독립이면 `λ = 1.0` (5%×5%/5%), 완전 동시 이동이면 `λ = 1/0.05 = 20`.  
실제로는 0~1 범위로 정규화: **랜덤 독립 기준값 = 0.05**, 높을수록 위기 시 동조화.

#### 출력 매트릭스

| 매트릭스 | 내용 | 활용 |
|---|---|---|
| 섹터 × 섹터 λ_L | 섹터 간 동시 폭락 확률 | 분산투자 효과 평가 |
| 섹터 × 섹터 λ_U | 섹터 간 동시 급등 확률 | 추세 추종 효과 |
| 섹터 × 매크로 λ_L | VIX·금리·달러와의 동시 하락 확률 | 헤지 수단 선별 |

#### 온톨로지 규칙 연결

`co_crash_cluster` 규칙:
- λ_L ≥ 0.40인 피어 섹터가 3개 이상이면 → `HEDGE` 시그널
- 의미: "이 섹터를 보유하면 위기 시 포트폴리오가 비선형으로 확대됨"

#### Layer 4(Cholesky)와의 직교 분업 (2026-06-08)

코퓰러(Layer 3, **꼬리** 의존성)와 콜레스키(Layer 4, **본체** 가우시안 의존성)는 서로 다른 시장 상태를 모델링한다:

| 레이어 | 측정 | 엣지 | 트리거 룰 |
|---|---|---|---|
| L3 Copula | λ_L (5% 하위 동시 발생률) | `CO_CRASH_WITH` (섹터⟷섹터, 대칭) | `co_crash_cluster` |
| L4 Cholesky | var_attr[i,j] (직교 충격 j 가 i 분산에 기여한 비율) | `CO_MOVES_WITH` (j→i, 비대칭) | `shock_propagator`, `variance_concentrated` |

평상시 가우시안 그래프(L4)와 위기 코퓰러 그래프(L3)가 동시 존재 — 룰셋이 두 그래프를 따로 본다.

### 4-5. Layer 4 — Multivariate MC + Cholesky 분산 분해 (NEW 2026-06-08)

**파일**: `macro_research/ontology/mvg_mc.py`

**목적**: 평상시(비위기) 가우시안 공동 변동성 구조의 *상관 휴리스틱* 분해. 코퓰러가 캡처
못 하는 본체 영역의 prior + 멀티에셋 시뮬 base rate 산출.

> **⚠ 인과 해석 금지 (2026-06-08 비평 반영)**: 본 레이어가 산출하는 `var_attr[i,j]`,
> `propagation_score`, `self_share` 는 모두 **콜레스키 변수 순서에 100% 조건부**.
> 섹터에 대한 PCMCI 인과 발견은 적용 불가하므로 순서는 `|corr|` 가중 eigenvector
> centrality 라는 *임의 휴리스틱* 으로 결정. 즉 "centrality 1위 = 시스템 충격원" 은
> *prior* 이지 발견된 인과가 아니며, SVAR 재귀식별의 잘 알려진 순서 의존성을 그대로
> 상속한다. shock_propagator / variance_concentrated 룰의 reasoning 도 동일.

#### 콜레스키 분산 분해

섹터 수익률 상관행렬 Σ를 콜레스키 분해:
```
Σ = L L^T,   L: 하삼각
```
정규성 가정 하:
```
Var(r_i) = Σ_k L[i,k]² · Var(z_k)   (z_k 독립 N(0,1))
var_attr[i, j] = L[i,j]² / Σ_k L[i,k]²    (행합 = 1)
```
→ `var_attr[i, j]` = "j 의 직교 충격이 i 분산에 기여한 비율".

#### 변수 순서 (Cholesky ordering)

콜레스키 분해는 **변수 순서 의존적** (SVAR 재귀식별과 동일). 본 모듈은:

1. PCMCI 인과 체인은 *매크로 변수* 간만 발견 → 섹터엔 직접 적용 불가
2. 대신 **|corr| 가중 그래프의 eigenvector centrality 내림차순** 으로 순서 결정
3. 가장 centrality 높은 섹터 = 시스템 충격원 → 첫 컬럼 → 자기 분산 100% self, 다른 섹터들이 잔차로 받음
4. fallback = 알파벳 순

```python
# 현재 mid_vix 시점 순서 (centrality 내림차순):
# XLI → XLB → XLY → XLF → XLRE → XLC → XLV → XLK → XLP → XLU → XLE
```

#### 핵심 노드 속성

각 SectorNode 에 콜레스키 결과 부착:

| 속성 | 의미 |
|---|---|
| `var_decomposition[j]` | j 의 직교 충격이 이 섹터 분산에 기여한 비율 (합=1) |
| `self_share` | var_decomposition[self] — 자기 직교 충격 비중 |
| `propagation_score` | 자기 충격이 *다른* 섹터들 분산에 기여한 합 (자기 제외) — 시스템 전파력 |
| `cholesky_order` | 0 = 가장 시스템적인 충격원 |

#### NORTA 멀티에셋 시뮬

다변량 GBM 경로 생성으로 시그널과 무관한 *시장 구조 base rate* 산출:

```python
r_t = μ_d + L · z_t · σ_d   (z_t ~ N(0,I))
```
- σ_d = 섹터별 GARCH 연환산 변동성 / √252
- horizon = 21영업일 (기본)
- n_sim = 5000

산출 통계 (`joint_drawdown_stats`):
- `p_any(τ)`: P(적어도 한 섹터 ≤ -τ)
- `p_half(τ)`: P(≥절반 섹터 ≤ -τ) — 시스템 동시 폭락 근사
- `p_all(τ)`: P(모든 섹터 ≤ -τ)
- `mean_hit_count`: 평균 동시 hit 섹터 수
- `top_pair_co_drawdown`: τ=5% 기준 상위 페어 공동 drawdown 확률

**현재 mid_vix 결과 (n_sim=5000, horizon=21d)**:

| τ | P(≥1 hit) | P(≥절반 hit) | P(전부 hit) |
|---|---|---|---|
| 3% | 97.2% | 39.6% | 1.4% |
| 5% | 85.6% | 11.7% | ~0% |
| 10% | 31.9% | 0.1% | 0% |

상위 동조 페어: XLY-XLK 21.6%, XLI-XLK 16.9%, XLK-XLE 15.8% (시장 베타 높은 쌍들).

이게 **시그널과 무관하게 시장 구조만으로 본 위험도** — 듀얼 전략 진입을 평가할 때 base rate 기준선.

### 4-8. IV_spread 신호 분해

**파일**: `macro_research/signal_decomposition.py`

온톨로지의 핵심 신호 `IV_spread`를 두 성분으로 분해하여 예측력의 원천을 파악:

```
IV_spread   = CAPM_IV − VIX/100
            = sqrt(β²·VIX² + σ_idio²) − VIX/100

beta_comp   = β × VIX/100        (시장 베타가 만드는 IV 기여분)
```

- `beta_comp`가 주도 → 해당 섹터가 시장 평균보다 높은 베타 노출 상태 (시스템 리스크)

### 4-10. 몬테카를로 시뮬레이션

**파일**: `research_mc_straddle.py`

**목적**: 이벤트 발생 전 현재 변동성 수준에서 스트래들의 수익 확률을 계산.

**로그정규 GBM (Geometric Brownian Motion)**:

```python
log_drift  = μ_daily − 0.5 × σ_daily²          # Itô 보정항
log_rets   = log_drift + σ_daily × z             # z ~ N(0,1)
price_rels = exp( cumsum(log_rets) )              # 항상 양수 보장
```

- 기존 산술 수익률 합산(`np.log1p(shocks)`) 방식은 σ가 크면 음수 가격 발생 → 로그정규 GBM으로 교체
- `N_SIM = 20,000` 경로 × `horizon = 120`일 → `(20000 × 120)` NumPy 행렬 단 한 번의 연산
- BLAS/SIMD 병렬화로 수 밀리초 내 완료

**핵심 결과 (zero-drift 가정)**:

| 연간 변동성 | E[PnL] | P(both hit) | 해석 |
|---|---|---|---|
| 20% | ~+1% | ~5% | 일반 장세, 수익 미미 |
| 30% | ~+5% | ~18% | 변동성 상승 구간 |
| 50% | ~+15% | ~42% | 이벤트 직전 수준 |
| 70% | ~+25% | ~60% | 위기 구간 |

Long Gamma 특성: 변동성이 클수록 E[PnL] 단조 증가. 드리프트=0 가정 시 break-even vol → 0.

#### 다변량 확장 — NORTA + Cholesky (2026-06-08, `ontology/mvg_mc.py`)

단일자산 GBM은 섹터 간 *공동* 변동을 보지 못한다 (호르무즈 발생 시 XLE 와 XLB 가 동시에 흔들리는 구조 미반영). 본 모듈은 NORTA(NORmal-To-Anything)로 확장:

```python
# 가우시안 공간에서 상관 주입 → 마진 정규 → (옵션) GPD 합성 역변환
z   = rng.standard_normal((n_sim, horizon, P))
g   = z @ L.T                              # L = chol(Σ_sector)
r_t = μ_d + g * σ_d                        # 섹터별 일별 로그수익률
```

- σ_d = `garch_vol.parquet` 최신 행 / √252 (섹터별 GARCH 연환산)
- L = `cholesky_decompose()` 의 하삼각행렬 (centrality ordering)
- 출력 = `(n_sim, horizon, P)` 누적 로그수익률 → drawdown 통계

**산출 통계는 4-5 (Layer 4) 참고**. 시그널과 무관한 base rate 로 듀얼 전략 진입의 기준선.

### 4-12. Causal Discovery — PCMCI

**파일**: `macro_research/causal_macro.py`

**목적**: 이론에 의존하지 않고 데이터에서 직접 거시 변수 간 인과 관계를 추론한다.

#### 왜 PCMCI인가

일반 그랜저 인과 검정은 다른 변수들의 공통 원인(common driver)을 통제하지 않아 허위 인과(spurious causality)가 발생한다. PCMCI(Peter-Clark Momentary Conditional Independence)는 다음을 동시에 수행한다:

1. **PC 알고리즘**: 각 변수의 과거값 중 독립적으로 유의한 부모(parent) 변수만 선별
2. **MCI 검정**: X→Y 링크를 검정할 때 X의 과거, Y의 과거, 모든 공통 원인을 조건부 통제
3. **결과**: 허위 상관이 제거된 DAG(방향 비순환 그래프), 각 엣지에 lag·계수·p값 포함

#### 파이프라인 5단계

```
Stage 1  데이터 수집
  yfinance: VIX, US10Y, US2Y, DXY, OIL  (2010-01 ~ 현재, 월별)
  FRED:     CREDIT_SPREAD(BAA10Y), CPI(CPIAUCSL), FED_FUNDS(FEDFUNDS)
        ↓
Stage 2a ADF 단위근 검정 → 비정상 시계열 1차 차분
Stage 2b VIF(분산팽창지수) 다중공선성 검사 (임계값=10)
        ↓
Stage 3  Change-point Detection
  인접 24개월 공분산 행렬 간 Frobenius 거리 → 피크 = 구조적 레짐 전환점
  (ruptures 대신 numpy/scipy 순수 구현 — Python 3.14 호환)
        ↓
Stage 4  PCMCI × 2
  전체 기간 (2010~ , ~194개월) + 현재 레짐 (마지막 changepoint 이후)
        ↓
Stage 5  Link 추출 + 안정성 분류
```

### 4-13. Kinetic Layer — Signal → Action (2026-06-08 신설)

#### vol 라우팅 수정 (2026-06-08 비평 반영)

비평: 시스템이 자기 엣지로 표명하는 `vol_overpriced`(VRP>2% = IV>RV = 옵션 비쌈)
신호를 키네틱 레이어가 ETF 현물 롱(OVERWEIGHT → +1)으로 라우팅했음 — 논리 모순.
스트래들은 *롱 옵션* 이라 IV 가 비싼 섹터에서 구조적으로 마이너스 EV (프리미엄 +
호가스프레드 + 세타 지불).

수정 (`trigger.py`):
- `DIRECTIONAL_EXCLUDE_RULES = {"vol_overpriced"}` — 방향성 라우팅에서 제외
- `SHORT_VOL_PENALTY = 0.5` — 같은 사이클에 `vol_overpriced` 가 발화한 섹터의
  STRADDLE 사이즈를 ×0.5 로 페널티 (`kinetic_executor.build_plan` 이 `size_penalty`
  를 scale 에 곱함). 후속으로 **숏 볼 트랙(short straddle / put credit spread)
  신설 예정** — 현재는 미구현 = **알려진 한계**.

검증 (2026-06-08 dry-run, mid_vix): XLY/XLC/XLF/XLRE 가 vol_overpriced LONG 으로
잘못 라우팅되던 게 사라지고, rate_beneficiary/rate_victim 만 남음 → XLRE/XLU/XLV
가 모두 SHORT. XLK/XLY/XLF/XLC/XLI 의 스트래들 qty 가 페널티로 절반 축소.

**핵심 아이디어**: 시그널당 readout 두 개(크기, 방향) → 학습 신호 *부분 상관*. 두 readout
은 완전 직교가 아니라 ρ ∈ (0,1) 의 부분 상관 — 큰 변동이 음(-) 방향으로 터지면 straddle
익절과 directional SHORT 가 동시에 맞는다. "두 배 학습 신호" 는 과장이며 실제 상관 ρ 를
attribution 의 cross-strategy 표로 측정해 두 트랙이 어느 정도 독립적인지 사후 검증할
것 (현재 표본 0건). 룰셋이 전략별로 다르다는 점은 여전히 유효: `rate_*` 는 스트래들에서
제외, `vol_overpriced` 는 **양쪽 모두에서 트리거 라우팅 제외** (2026-06-08 수정 — 아래
"vol 라우팅 수정" 참고).

#### 사이징 공식

```
straddle_budget   = equity × BASE_WEIGHT(3%) × scale(conf/0.65, cap 2x) × type_w × rule_mult
directional_notion = equity × DIR_BASE_WEIGHT(5%) × scale × type_w × rule_mult
qty = floor(budget / cost_per_unit)   # 최소 1

type_w     = {OVERWEIGHT:1.0, UNDERWEIGHT:1.0, HEDGE:1.0, MONITOR:0.3}
rule_mult  = rule_performance.json[f"{rule}|{strategy}"]   # feedback 루프 산출, 기본 1.0
```

`MONITOR ×0.3` 의 의도: mid_vix 에선 발화 룰이 `causal_chain_monitor` 일변도라 빈도를 유지하되 노출은 축소. 피드백이 실현 성과로 prior 를 보정.

#### 청산 조건

| 전략 | 익절 | 손절 | 시간 통제 |
|---|---|---|---|
| STRADDLE (옵션) | +25% | 없음 (롱 볼 비대칭) | DTE ≤ 10일 강제 청산 |
| DIRECTIONAL (현물) | +10% | -7% | 최대 보유 21영업일 강제 |

방향성은 시간을 우호적으로 두지 않음 — "설명의 예측 지평(~수주) 내에서 채점" 원칙.

#### Trade Journal (append-only JSONL)

`output/trade_journal.jsonl` 한 줄당 1 이벤트:

```json
// ENTRY
{"event":"entry", "trade_id":"XLI-2026-06-08-a1b2c3", "strategy":"directional",
 "ticker":"XLI", "rule":"shock_propagator", "all_rules":["shock_propagator"],
 "signal_type":"HEDGE", "confidence":0.85, "regime":"high_vix",
 "direction":-1, "spot":174.20, "qty":37, "entry_cost":6445, "reasoning":[...]}

// EXIT (같은 trade_id 로 매칭)
{"event":"exit", "trade_id":"XLI-2026-06-08-a1b2c3", "strategy":"directional",
 "exit_reason":"손절", "exit_value":5994, "pnl":-451, "pnl_pct":-0.07, "holding_days":4}
```

미청산 매칭 키 = `(ticker, strategy)` — 같은 티커에 스트래들과 현물을 동시 보유할 수 있어 필요.

#### 최소 사이즈 모드 (`--minimal`, 2026-06-08)

첫 RTH 사이클의 실거래 검증용. 사이즈 1/6, 시그널 1/3 로 축소.

| 모드 | BASE_WEIGHT | DIR_BASE_WEIGHT | MAX_SIGNALS |
|---|---|---|---|
| 통상 | 3% | 5% | 6 |
| `--minimal` | 0.5% | 0.5% | 2 |

목적은 P&L 검증이 아니라 **trade_journal 첫 채우기 + 호가스프레드/세타 실측치 수집**.
표본이 N≥10 누적된 후에야 stop-rule 과 사이징 보정이 의미를 갖는다.

#### Attribution + 사후 사이징 보정 (구 "피드백 루프")

```
trade_journal.jsonl
        ↓
attribution.build_attribution()
  by_strategy        : 전략 단위 비교 (straddle vs directional)
  by_rule_strategy   : (rule × strategy) 합성키 — 부분상관 채점의 핵심
  by_rule            : 룰 단위 (전략 합산)
  by_regime          : 레짐별
  by_signal          : signal_type 별
        ↓
feedback.build_feedback()  (2026-06-08 보수화 + stop-rule)
  per (rule|strategy):
    n < 5                                    → mult = 1.0  (표본부족, prior 유지)
    n ≥ 10 AND avg_pnl_pct < -5%             → mult = 0    (영구 차단, stop-rule)
    그 외:
      factor = min(1, n / 15)
      mult = clip(1.0 + avg_pnl_pct × factor, 0.7, 1.3)
        ↓
output/rule_performance.json
        ↓
kinetic_executor 다음 사이클이 자동 반영. mult=0 인 키는 진입 자체 스킵.
```

**범위**: 본 모듈이 보정하는 것은 *사이징 승수*뿐. `inference.py` 의 임계값 T 와 청산
임계값은 **수동 튜닝** 으로 남김 — 어떤 시그널이 발화하는지를 자동으로 바꾸지 않음 (안전성).

**⚠ "학습 루프" 가 아니라 "사후 사이징 보정"** (2026-06-08 비평 반영): 핵심 룰
(high_vix 발화: natural_hedge / crash_vulnerable / fat_tail_alert) 은 표본 누적이
수년 단위라 사실상 **열린 루프**. shrinkage 만으론 신호를 만들지 못한다. 보정이 실제로
의미를 갖는 건 mid-frequency 발화 룰(causal_chain_monitor, rate_*) 뿐. stop-rule
(`mult=0` 영구 차단) 은 명백히 손실 누적된 (rule, strategy) 키를 은퇴시켜 70% 사이즈
드래그를 막는다.

**부분상관 학습의 예**: 같은 `causal_chain_monitor` 룰이 (`|straddle`) 에선 +25% 익절 자주,
(`|directional`) 에선 방향 자주 틀림 → 두 승수가 반대 방향으로 움직임 → "이 룰은 크기는
맞히지만 방향은 못 맞힌다" 가 정량 학습. 다만 두 readout 이 완전 직교는 아니므로 표본
부족 시 노이즈 학습 위험 → `MIN_TRADES=5`, `clip(0.7, 1.3)` 으로 보수화 (2026-06-08 수정,
이전 `MIN_TRADES=3`, `clip(0.5, 1.5)`).

### 4-14. 백테스트 모드 (NEW 2026-06-10)

**파일**: `macro_research/backtest_pipeline.py`

#### 설계

| 항목 | 값 |
|---|---|
| 기간 | END_DATE − 180일 ~ END_DATE (~126 거래일) |

- **GARCH IV proxy**: 진짜 옵션 IV 아님 (라운드 4 비평 그대로). 백테스트 옵션 가격은
